# 账号登录注册联调清单

本文档用于验证当前账号体系的真实业务链路，覆盖：

- 微信登录
- 手机号验证码注册/登录
- 账号密码登录
- 已登录用户绑定手机号
- 手机号绑定触发自动并账号
- 手机号验证后的下载权限

## 一、环境准备

### 1. 后端配置

本地联调建议使用短信 mock：

```env
TENCENT_SMS_MOCK_ENABLED=true
TENCENT_SMS_MOCK_CODE=123456
TENCENT_SMS_EXPOSE_MOCK_CODE=true
```

说明：

- `send-code` 接口会返回 `mock_code`
- 小程序登录页和手机号绑定页会自动把 mock 验证码回填到输入框
- 本地不用真实腾讯云短信配置也能跑通链路

### 2. 清理测试数据建议

在开始联调前，准备三类测试账号：

1. 纯微信账号
2. 纯手机号账号
3. 微信账号和手机号账号各自独立，后续用来验证自动并账号

## 二、登录注册联调顺序

### 场景 A：手机号首次注册/登录

预期目标：

- 手机号未存在时，自动创建新账号
- 自动绑定手机号身份
- 返回同一个 `user_id` 的 token

步骤：

1. 打开登录页，切换到“手机号登录”
2. 输入一个未使用过的手机号
3. 点击“获取验证码”
4. 确认页面自动回填 `123456`
5. 点击“手机号注册/登录”

验证点：

- 登录成功
- `userInfo.phone` 已写入本地缓存
- 后端 `user_identities` 新增一条 `phone` 身份
- 新用户 `user_profiles.phone` 不为空

### 场景 B：手机号二次登录

预期目标：

- 不新建账号
- 命中原手机号账号

步骤：

1. 退出登录
2. 再次用同一手机号走验证码登录

验证点：

- 登录成功
- 返回的 `id` 与第一次手机号登录的 `id` 一致

### 场景 C：微信首次登录

预期目标：

- 未命中现有微信身份时，创建微信账号
- 微信身份写入统一身份表

步骤：

1. 打开登录页，切换到“微信登录”
2. 点击“微信一键登录/注册”

验证点：

- 登录成功
- 返回 `id / token`
- `user_identities` 写入 `wechat_openid` / `wechat_unionid`

### 场景 D：账号密码登录

预期目标：

- 使用已有用户名密码正常登录
- 返回同一 `user_id`

步骤：

1. 先在“个人资料 / 账号安全”页设置用户名密码
2. 退出登录
3. 切换到“账号登录”
4. 输入用户名密码登录

验证点：

- 登录成功
- 返回 `id` 与设置密码的账号一致

## 三、手机号绑定联调

### 场景 E：已登录用户绑定一个未被占用的手机号

预期目标：

- 绑定成功
- 当前账号不变

步骤：

1. 用微信账号登录
2. 打开“设置 -> 绑定手机号”
3. 输入一个未被占用的手机号
4. 获取验证码
5. 提交绑定

验证点：

- 提示“手机号绑定成功”
- `profile.phone` 刷新后有值
- 设置页显示手机号已绑定
- `user_identities` 新增手机号身份

### 场景 F：已登录用户绑定一个已被别的账号占用的手机号

预期目标：

- 不新增账号
- 自动把当前账号并到手机号对应账号
- 业务资产迁移到主账号

步骤：

1. 准备一个纯微信账号 A
2. 准备一个纯手机号账号 B
3. 登录账号 A
4. 在绑定手机号页输入账号 B 的手机号
5. 提交绑定

验证点：

- 返回 `merged_to_user_id`
- 前端弹窗提示“已自动完成账号归并”
- 前端刷新资料后，显示为目标账号信息
- 源账号 `merge_status = merged_source`
- 主账号 `merge_status = merged_target`
- 关键资产已迁移

## 四、并账号资产迁移验证

需要重点核验这些表：

- `users.stones`
- `user_orders`
- `stone_records`
- `ai_tasks`
- `ai_video_tasks`
- `templates.creator_user_id`
- `template_comments`
- `template_shares`
- `template_likes`
- `template_unlocks`
- `designer_follows`
- `designer_reviews`
- `invite_relations`
- `user_memberships`
- `user_invite_codes`
- `user_profiles`
- `certification_applications`
- `code_sessions`
- `user_identities`

重点验证规则：

1. 主账号余额 = 主账号余额 + 源账号余额
2. 源账号余额归零
3. 主账号保留更强的 `can_withdraw`
4. 主账号保留有效密码哈希
5. 点赞/解锁/关注关系不产生重复记录
6. 邀请关系不会变成“自己邀请自己”
7. 会员权益取更优的一份
8. 会话映射指向主账号

## 五、手机号验证后的权限验证

### 场景 G：模板下载权限

预期目标：

- 手机号验证通过后，模板下载相关接口正常放行

步骤：

1. 用已验证手机号的账号登录
2. 打开模板详情
3. 尝试下载模板图片

验证点：

- `user_phone_verified = true`
- 下载接口不再报“请先完成手机号授权验证”

### 场景 H：AI 原图/视频下载权限

预期目标：

- 手机号验证通过后，AI 原图/视频下载接口放行

步骤：

1. 用已验证手机号的账号登录
2. 进入 AI 任务详情
3. 尝试下载原图 / 视频

验证点：

- 接口返回真实下载地址
- 不再卡在手机号验证提示

## 六、当前已知限制

1. 全量 `go test ./...` 在本机 route 层仍受外部环境问题影响，当前以 `go test ./model ./component ./processor` 为主验证。
2. 现在资料层 `phone` 已作为正式手机号字段引入，但旧的 `enterprise_wechat_contact` 仍保留兼容。
3. 生产环境切换真实腾讯云短信时，需要关闭 mock：

```env
TENCENT_SMS_MOCK_ENABLED=false
TENCENT_SMS_EXPOSE_MOCK_CODE=false
```

同时补齐：

- `TENCENT_SMS_SECRET_ID`
- `TENCENT_SMS_SECRET_KEY`
- `TENCENT_SMS_APP_ID`
- `TENCENT_SMS_SIGN_NAME`
- `TENCENT_SMS_LOGIN_TEMPLATE_ID`
- `TENCENT_SMS_BIND_TEMPLATE_ID`
