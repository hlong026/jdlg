# 手机号注册登录 + 会员下载限制 — 实施方案

> 最后更新：2026-04-24

---

## 一、功能概览

### 1. 手机号验证码登录/注册
- 新增「手机登录」方式，与现有微信登录、账号密码登录并列
- 输入手机号 → 获取验证码 → 登录（已注册）或注册（新用户）
- 新用户注册后弹窗选择身份类型（业主/设计师/施工队/企业），仅标签，不影响权限
- 所有登录方式通过 `user_identities` 表统一绑定到同一用户 ID

### 2. 手机号绑定
- 已登录用户可在「个人中心」绑定手机号
- 绑定时验证短信验证码
- 若手机号已被其他用户使用，返回冲突提示（code=40901）

### 3. 会员下载限制
- 不同会员套餐配置不同的下载限制参数
- 限制维度：下载有效期、累计下载量、每日下载上限、频控
- 模板下载接口中自动检查限制

### 步骤 2：腾讯云短信配置

**目标**：配置短信服务环境变量，验证验证码发送

**前置条件**：需要腾讯云 SMS 服务已开通，获取以下信息：
- SecretID
- SecretKey
- SdkAppID
- 签名名称（SignName）
- 模板 ID（TemplateID）

**操作**：
1. 设置环境变量（或配置文件）：
```bash
export TENCENT_SMS_SECRET_ID="你的SecretID"
export TENCENT_SMS_SECRET_KEY="你的SecretKey"
export TENCENT_SMS_SDK_APP_ID="你的SdkAppID"
export TENCENT_SMS_SIGN_NAME="你的签名"
export TENCENT_SMS_TEMPLATE_ID="你的模板ID"
```

2. 启动服务，调用发送验证码接口：
```bash
curl -X POST http://localhost:端口/api/miniprogram/sms/send \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000"}'
```

**期望结果**：
- 返回 `{"code":200,"msg":"验证码发送成功"}`
- Redis 中存入 `sms:code:13800138000`（5 分钟有效）
- 控制台输出短信发送日志

**开发模式**：如果环境变量未配置，验证码仅打印到控制台日志，不实际发送短信。日志格式：
```
[SMS Dev Mode] phone=13800138000 code=123456
```

**验证码 Redis Key 说明**：
| Key | TTL | 用途 |
|-----|-----|------|
| `sms:code:{phone}` | 5 分钟 | 验证码本身 |
| `sms:limit:{phone}` | 60 秒 | 发送冷却期 |
| `sms:daily:{phone}:{date}` | 到当天结束 | 每日发送计数（上限 10） |
| `sms:fail:{phone}` | 5 分钟 | 验证失败计数（5 次后失效） |

### 步骤 6：会员下载限制

**目标**：验证不同会员套餐的下载限制正确生效

#### 6.1 配置套餐数据

在管理后台「套餐管理」中，为各套餐设置下载限制参数：

| 套餐 | 下载有效期(天) | 累计可下载 | 每日上限 | 频控(次/分钟) |
|------|-------------|-----------|---------|-------------|
| 普通会员 | 7 | 10 | 3 | 3 |
| 白银会员 | 30 | 80 | 10 | 5 |
| 黄金会员 | 90 | 200 | 20 | 8 |
| 钻石会员 | 0(无限) | 0(无限) | 0(无限) | 0(无限) |

#### 6.2 限制检查逻辑

下载模板时，后端检查链路：
```
1. 查用户是否有活跃会员 → 无会员则走原有扣费逻辑
2. 有会员 → 查套餐的下载限制参数
3. 检查下载有效期（granted_at + download_validity_days > now?）
4. 检查累计下载量（COUNT template_unlocks WHERE user_id=x <= max_total_downloads?）
5. 检查每日上限（Redis 计数器 template:dl:{uid}:{date} <= daily_download_limit?）
6. 检查频控（Redis 计数器 template:rl:{uid}:{minute} <= rate_limit_per_minute?）
7. 全部通过 → 放行下载 + RecordDownload 更新计数器
8. 任一超限 → 返回对应提示
```

#### 6.3 测试场景

```bash
# 测试下载（需要会员用户 token）
curl -X POST http://localhost:端口/api/miniprogram/templates/模板ID/download \
  -H "Authorization: Bearer 会员用户token"
```

**超限返回示例**：
```json
{"code": 40301, "msg": "下载次数已用完"}
{"code": 40301, "msg": "今日下载次数已达上限"}
{"code": 40301, "msg": "操作过于频繁，请稍后再试"}
{"code": 40301, "msg": "下载有效期已过"}
```

**Redis Key 说明**：
| Key | TTL | 用途 |
|-----|-----|------|
| `template:dl:{uid}:{date}` | 到当天结束 | 每日下载计数 |
| `template:rl:{uid}:{minute}` | 60 秒 | 频控计数 |

## 七、注意事项

1. **手机号格式**：仅支持 11 位中国大陆手机号（正则 `^1\d{10}$`）
2. **验证码安全**：一次性使用，验证成功后自动删除；5 次验证失败后验证码失效
3. **开发模式**：SMS 环境变量未配置时，验证码打印到服务日志，方便本地调试
4. **下载限制**：仅对有活跃会员的用户生效，非会员走原有扣灵石逻辑
5. **手机号去重**：上线前需先执行 `sql/dedup_phones.sql` 清理已有重复数据
6. **账号合并**：`/phone/bind` 发现冲突时使用 `GetEffectiveUserByID` 判断是否已合并到当前用户
7. **Token 传递**：小程序接口统一使用 `token` 请求头传递（不是 Authorization Bearer）

```bash
# 检查腾讯云 SMS 配额
```

#### 手机号登录返回 "验证码错误" 但验证码正确
```bash
# 检查 Redis 中验证码是否存在
redis-cli GET "sms:code:$TEST_PHONE"

# 如果 >= 5，说明验证码已被失效，需要重新发送
```

#### 手机绑定返回 40901
```bash

$MYSQL_CMD -e "

"
# 如果对方用户已 merge 到当前用户，应该放行
# 如果不是，说明确实被另一个独立用户占用了
```
